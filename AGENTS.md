# 서보 브리지 개발 지침

## PCA9685 출력 요구사항

- 기존 시리얼 명령 프로토콜과 채널 번호 체계를 유지한다.
- cosine 모션 보간, 채널별 독립 수행시간, 지연 실행 및 마지막 데드밴드
  보정 단계를 유지한다.
- PCA9685 PWM 설정 주파수는 200 Hz를 유지한다.
- 25 MHz 발진기와 200 Hz 설정의 실제 분주 주파수를 기준으로 마이크로초
  단위 펄스 폭을 PCA9685 카운터 값으로 변환한다.
- 데드밴드와 마지막 보정은 실제 PCA9685 카운터 단위로 판단한다.
- 채널별 가동 범위, 방향, 오프셋, 초기 위치 및 각도 제한 설정을 유지한다.
- 부팅과 `#RESET`은 모터 8번부터 1번까지 역순으로 초기화하며 각 모터의
  이동 시작 간격은 1000ms이다. 각 이동은 기존 cosine 보간과 속도 제한을
  유지한다.
- 부팅 시 PCA9685 출력을 먼저 끄고 역순으로 초기 PWM을 인가한다. 마지막
  1번 모터의 초기 PWM 인가가 시작된 뒤 `READY`를 출력하며, 그전에는
  시리얼 명령을 처리하지 않는다.
- `#RESET`은 진행 중인 모션과 모든 지연 명령을 현재 계산 위치에서 취소한
  뒤 순차 리셋을 예약하고 `OK`를 즉시 반환한다.
- `#RESET`으로 예약된 각 모터가 실제로 시작될 때
  `RESET motor8 angle:90.00` 형식의 메시지를 8번부터 1번까지 출력한다.
  부팅 초기화와 일반 지연 명령에서는 이 메시지를 출력하지 않는다.
- 일반 루프의 모션 갱신 시각은 시리얼 명령 처리가 끝난 뒤 읽는다. 새 명령의
  모션 시작 시각보다 오래된 `millis()` 값으로 갱신하여 unsigned 시간 계산이
  언더플로되지 않도록 유지한다.
- PCA9685 한 개로 최대 16개 채널을 지원한다. 축을 늘릴 때는
  `SERVO_COUNT`, `SERVO_CHANNELS` 및 모든 축별 설정 배열을 함께 늘린다.
- PCA9685 채널 0은 사용하지 않는다. 모터 번호와 PCA9685 채널 번호는
  항상 동일하게 유지하며 단일 보드에서는 모터 1~15를 사용할 수 있다.
- 오류 응답은 `ERR,<영문 코드>,<한국어 설명>,INPUT=<입력>` 형식을
  유지하고 시리얼 클라이언트는 UTF-8로 디코딩한다.
- 일반 각도 및 Raw PWM 명령이 정상 등록되면 `OK` 다음 줄에 선택된 모터를
  번호순으로 출력한다. 각도 항목은 입력 각도와 함께 PCA9685 카운터로
  양자화한 목표 펄스 및 카운트를 출력한다. 예:
  `motor1:angle 100.00,pulse 1425.72us,count 1150,duration 2.000s`.
  또는 `motor1:pulse 1500.00us,duration 1.000s,delay 0.800s` 형식이며
  여러 모터는 `/`로 구분한다. `duration`은 요청시간과 안전 최소시간 중
  큰 접수 시점 기준 예정 실행시간을 사용한다. 지연 실행 시 시작 위치가
  달라지면 실제 시작 과정에서 안전 최소시간이 더 늘어날 수 있다.
- 기본 포지션은 `SERVO_POSITIONS` const 2차원 배열에 모든 축의 논리각을
  저장한다. `#GOPOS1T1000` 형식으로 선택하며 포지션 번호는 1부터
  `SERVO_POSITION_COUNT`까지이다. `T`를 생략하면
  `SERVO_POSITION_DEFAULT_TIME_MS`의 2000ms를 사용하고, 지정할 때는
  100~9999ms 범위여야 한다.
- `#GOPOS`는 모든 모터에 같은 요청시간을 적용하지만 각 모터의 안전
  최소시간 자동 연장은 유지하며, 정상 접수 시 일반 각도 명령과 같은 상세
  응답을 출력한다.
- 테스트 모터 선택 명령은 공백 없는 `#TEST<모터번호>` 형식이다.
  `#TEST 5`처럼 입력하면 `TEST_FORMAT` 오류와 올바른 `#TEST5` 예시를
  반환한다.

## 오프셋 저장 요구사항

- 축별 오프셋은 런타임 배열 `servoOffsetDeg`를 사용하고 하드코딩된
  `DEFAULT_SERVO_OFFSET_DEG`는 NVS 저장값이 없거나 유효하지 않을 때만
  사용한다.
- `#OFFSET#1A-9.45#3A2.7` 형식으로 여러 모터의 오프셋을 한 번에 설정한다.
- 정확히 `#OFFSET`만 입력하면 저장 없이 현재 오프셋을 조회한다. `OK`를
  먼저 출력하고 다음 줄에 모든 모터의 오프셋을 설정 성공 응답과 같은
  형식으로 출력하며, NVS 기록이나 서보 이동을 수행하지 않는다.
- 오프셋 명령은 모든 항목을 검증한 뒤 한 번에 적용하고 저장한다. 일부
  항목만 적용하거나 부분 저장하지 않는다.
- 오프셋 허용 범위는 -180도부터 180도까지이며 단위는 degree이다.
- 오프셋 적용 후 보정각은 축의 실제 물리 가동범위로 제한한다. 180° 축은
  `0~180°`, 270° 축은 중앙 구간 양쪽 45° 여유를 포함한 `-45~225°`를
  허용하여 최종 PWM 500~2500us 범위를 사용할 수 있게 한다.
- ESP32 Preferences의 `servo-bridge` namespace와 `offsets` key에 전체
  float 배열을 blob으로 저장하고 부팅 시 불러온다.
- 오프셋 저장 명령은 서보를 즉시 움직이지 않는다. 이후 각도 명령과 다음
  부팅 초기 위치 계산부터 새 오프셋을 적용한다.
- 저장 성공 시 기존 `OK`를 먼저 출력하고 다음 줄에
  `angle offset => motor1:-4.00,...,motor8:0.00` 형식으로 모든 모터의
  현재 오프셋을 소수점 두 자리까지 출력한다.
- 오프셋 저장 실패나 명령 오류 시에는 전체 오프셋 목록을 출력하지 않는다.
- Raw PWM `P` 명령에는 오프셋을 적용하지 않는다.
- 저장 구조나 `SERVO_COUNT`를 변경할 때는 기존 blob 크기가 맞지 않으면
  기본 오프셋으로 안전하게 복귀하도록 유지한다.

## 하드웨어 요구사항

- 모든 서보의 합산 스톨 전류를 감당할 수 있는 별도 전원으로 서보를
  구동한다.
- ESP32-C3 보드에서 서보 전원 레일에 전원을 공급하지 않는다.
- ESP32-C3, PCA9685 및 서보 전원 공급 장치의 GND를 공통으로 연결한다.
- PCA9685 `VCC`는 ESP32-C3의 3.3V, `V+`는 별도 서보 전원에 연결한다.
- 현재 `OE`는 GND에 고정하므로 ESP32 전원 차단 시 서보 동작을 보장하지
  않는다.
- 서보 전원을 켜기 전에 PCA9685 로직 전압, I2C 풀업 저항 및 보드 주소를
  확인한다.
